安全防护概览

Tea API 安全设置提供了全面的安全防护机制，专门针对恶意攻击进行防护。

🎯 针对 1.txt 脚本的防护效果

  • 高频请求检测：0.05秒间隔的请求会被立即识别并阻止
  • 超长Prompt拦截：50,000+字符的随机内容会被拦截
  • 流式滥用防护：慢速读取的流连接会被强制断开
  • 自动IP封禁：检测到恶意行为后自动将IP加入黑名单

功能特性

🛡️异常行为检测

检测超长Prompt、随机内容、高频请求等异常行为，并进行实时评分和拦截。

⚙️请求大小限制

限制请求体大小、消息数量和内容质量，防止超大请求攻击。

📊流保护系统

监控流式请求，限制并发数量，检测慢客户端攻击。

🔒IP黑名单管理

自动和手动管理IP黑名单，支持临时和永久封禁。

📋安全日志审计

详细记录所有安全事件，支持筛选和分析。

API接口

GET /api/security/stats - 获取安全统计信息
GET /api/security/config - 获取安全配置
PUT /api/security/config - 更新安全配置
POST /api/security/blacklist - 添加IP到黑名单
DELETE /api/security/blacklist/:ip - 从黑名单移除IP

示例请求

// 获取安全统计 fetch('/api/security/stats', { headers: { 'Authorization': 'Bearer your-token' } }) .then(res => res.json()) .then(data => console.log(data)); // 更新安全配置 fetch('/api/security/config', { method: 'PUT', headers: { 'Authorization': 'Bearer your-token', 'Content-Type': 'application/json' }, body: JSON.stringify({ abnormal_detection: { enabled: true, max_prompt_length: 50000 } }) });

使用演示

1. 启用安全防护

在设置页面中选择"安全设置"，即可看到完整的安全配置界面。

2. 配置异常检测

// 异常检测配置示例 { "enabled": true, "max_prompt_length": 50000, "max_random_char_ratio": 0.8, "suspicious_score_limit": 100, "auto_blacklist_enabled": true }

3. 监控安全状态

通过安全概览页面可以实时查看：

  • 安全评分和状态
  • 被阻止的请求数量
  • 恶意行为检测次数
  • 黑名单IP数量
  • 活跃流连接数

4. 管理IP黑名单

可以手动添加或移除IP，查看封禁原因和时间，支持临时和永久封禁。
